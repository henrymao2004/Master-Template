\begin{abstract}
LLM agents increasingly improve themselves by distilling successful
interactions into reusable skills. This stores experience as persistent
behavioral policy, yet existing evaluations measure whether evolution improves
capability without testing what happens when success depends on an unsafe
shortcut. We identify \emph{skill misevolution}: a transient unsafe success is
extracted into an apparently legitimate skill and later reused on benign tasks
after the original attack has disappeared. We introduce \gym, a
task-source-agnostic environment that turns executable agent tasks into stateful
skill-learning episodes. It standardizes skill writing, versioned library
state, retrieval, replay, and fresh-session evaluation across agent frameworks
and evolution methods. On top of this environment, \bench defines an
exposure-to-carryover protocol, evolution comparison settings, and nine metrics for
online behavior, the evolved artifact, and post-attack behavior. We instantiate the benchmark with
AgentHazard as a source of harmful objectives, executable environments, and its
harmful judge. An offline AHA autoresearch stage discovers vulnerability
concepts suited to skill misevolution; we instantiate each concept into matched
malicious, benign, and persistence tasks and add a benign-completion judge.
Across four agent
frameworks using a shared MiniMax backbone, six
evolution methods and ten AgentHazard risk categories, benign utility,
malicious ASR, and benign-task contamination are
[BU/MA/BA]. Content Unsafety (CU), Unsafe Generalization (UG), and
Stealthiness (Stealth) scores are [CU/UG/Stealth]; unsafe retrieval reaches [URR],
carryover ASR reaches [CA], and carryover utility remains [C-Util]. Unsafe
content appears in the skill library before it consistently manifests as
behavior, and the pattern appears by round [R], after [K] malicious tasks. We
further introduce \safeevolve, a governance layer that audits and
repairs candidate skills, preserves verification steps, tracks provenance, and
retires skills with unsafe reuse histories. Under \safeevolve, the online
metrics are [BU'/MA'/BA'], artifact scores are [CU'/UG'/Stealth'],
and post-attack URR / carryover ASR / utility are
[URR' (or N/A when no unsafe skill is exported)/CA'/C-Util'].
These results show that self-improvement is a security-relevant policy update:
practice can coincide with greater capability and persistent unsafety.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

LLM agents are beginning to improve through use. Modern CLI and coding agents
can summarize successful trajectories, write or revise reusable skills, and
retrieve those skills when related tasks appear. This capability changes an
agent from a fixed policy with transient context into a system whose behavior is
partly rewritten by its own experience. Skill evolution is therefore more than
memory: it is a persistent policy update expressed as an editable artifact.

The signal used to produce that update is usually task success or efficiency.
Successful behavior, however, is not necessarily safe behavior. An agent may
complete a task by skipping a verification step, assuming authorization,
treating an irreversible action as routine, or following an attacker-controlled
observation. When such a shortcut appears in a successful trajectory, an
evolution method may preserve it together with the useful procedure.

This motivates a temporal safety question that independent-task evaluations cannot
observe. The original attack may disappear with the context window, but the
lesson extracted from it can remain in the skill library. A later benign task
can retrieve the skill, apply its over-generalized rule, and reproduce the harm
without a malicious instruction. A fresh session can inherit the same behavior
by loading the evolved library alone. We call this process \emph{skill
misevolution}: transient unsafe success becomes persistent, retrievable, and
generalizable policy.

Measuring misevolution requires an environment that exposes persistent learning
as experimental state. We introduce \gym, which wraps executable agent tasks
with a standardized skill lifecycle: it snapshots the
library before and after each three-task block, resets conversational and
workspace state between tasks, and records skill writes and retrievals. For the
final persistence probe, it reloads retrieval in a new executor using only the
frozen \texttt{SKILL.md}. Only this serialized skill artifact crosses into the
fresh session.

\bench is the evaluation protocol built on this gym. Its basic unit is a
21-task episode with three rounds of three malicious tasks followed by three
benign tasks, then three persistence probes in a fresh executor session.
No-evolution and benign-only-update controls provide reference
distributions for later harm under the base agent, benign learning, and learning
histories that include malicious experience.

The benchmark evaluates three stages of the failure. Online behavior is measured
by benign utility, malicious ASR, and benign ASR
(contamination). The evolved artifact is measured by 1--5 Content
Unsafety (CU), Unsafe Generalization (UG), and Stealthiness (Stealth) scores. Post-attack behavior is
measured by Unsafe Retrieval Rate (URR), carryover ASR, and carryover utility in
a clean session. An unsafe skill can remain latent
until a later task supplies the right retrieval cue. Behavior-only evaluation
therefore misses risk that has already entered the persistent policy.

Constructing this evaluation requires more than replaying harmful benchmark
prompts. We use AHA autoresearch~\citep{mao2026aha} offline to discover
falsifiable vulnerability concepts whose enabling conditions can be expressed
as reusable procedures. An AHA run-pair instantiator then constructs a
run pair on a shared task surface: an $M$ task exercises the unsafe mechanism,
while a $B$ task preserves the legitimate objective but removes the malicious
condition. Concept-matched $P$ tasks probe later reuse without a new attack.
AgentHazard supplies the harmful objective, executable environment, and harmful
trajectory judge; this work contributes the paired benign and persistence
tasks, the benign-completion judge, and the longitudinal episode assembly.

Our evaluation asks four questions. \textbf{RQ1} compares unsafe behavior and
utility with and without evolution. \textbf{RQ2} follows the
exposure-to-persistence sequence and tests whether unsafe artifacts, retrieval,
and later harm occur in the expected temporal order. \textbf{RQ3} measures how the observed pattern varies across evolution
methods, agent configurations, vulnerability concepts, task surfaces, exposure
budgets, and exposure timing at a fixed budget. \textbf{RQ4}
tests whether \safeevolve retains learning utility while showing lower risk
across the tracked sequence.

\safeevolve acts at the learning boundary rather than as another runtime
refusal layer. It audits the complete candidate skill bundle before admission,
repairs unsafe generalization instead of deleting useful procedures, preserves
verification requirements during revision, records the experiences from which
each skill was learned, and downweights or retires skills whose reuse is
followed by unsafe behavior. These components govern the persistent object
associated with the observed incremental risk: the evolved skill library.

Our contributions are:
\begin{enumerate}
  \item We identify \textbf{skill misevolution}, a temporal pattern in which
  transient unsafe success is recorded in a persistent skill and unsafe
  behavior reappears on benign tasks without the original attack.

  \item We introduce \gym, a reusable environment for safety evaluation of
  self-improving agents. It provides a common skill lifecycle, versioned library
  state, isolation and replay, retrieval traces, and adapters for multiple agent
  frameworks and evolution methods.

  \item We introduce \bench, a task-source-independent temporal protocol with
  evolution comparison settings and nine absolute metrics: benign utility,
  malicious ASR, benign ASR (contamination), three artifact scores (Content
  Unsafety (CU), Unsafe Generalization (UG), and Stealthiness (Stealth)), Unsafe
  Retrieval Rate (URR), carryover ASR, and carryover utility. We
  instantiate it through an AHA-guided construction pipeline that turns
  AgentHazard harmful-task contracts into concept-matched malicious, benign,
  and persistence tasks with a new benign-completion judge.

  \item We provide a systematic evaluation across four agent frameworks using
  a shared MiniMax backbone, six evolution methods, ten
  AgentHazard risk categories, AHA vulnerability concepts, task surfaces,
  exposure budgets, and timing placements, separating
  artifact-level latent risk from post-attack realized harm.

  \item We introduce \safeevolve and evaluate whether safety-aware skill
  admission, revision, provenance, and retirement attain lower observed risk
  while retaining learning utility.
\end{enumerate}

\section{Related Work}
\label{sec:related-work}

\paragraph{Self-improving agents and skill evolution.}
Position this work against methods that extract, edit, select, vote over, or
retrieve reusable skills from agent trajectories. Organize the literature by
the persistent artifact each method produces and the success signal used to
select it. The key gap is that these methods report capability improvement but
do not measure safety patterns across an evolving skill history.

\paragraph{Safety of persistent agent memory.}
Discuss memory poisoning, long-term memory evolution, retrieval attacks, and
forgetting. Distinguish a skill from a stored observation: a skill is an
executable or prescriptive procedure intended to generalize across tasks. The
failure studied here is endogenous policy formation from the agent's own
successful trajectory, not only storage of attacker-supplied text.

\paragraph{Executable safety-task sources.}
Agent safety benchmarks supply tasks, environments, attack inputs, and outcome
judges that a learning environment can execute. AgentHazard provides 2,653
computer-use instances, ten risk categories, multi-step attack strategies, and
a harmful trajectory judge~\citep{feng2026agenthazard}. These assets define the
harmful task contract in our instantiation; they do not provide benign twins,
persistence probes, or a benign utility metric. We construct those components
with AHA-guided vulnerability concepts~\citep{mao2026aha}, then use \gym to add
persistent skill state, lifecycle instrumentation, and cross-task execution.

\paragraph{Governance of evolving libraries.}
Contrast \safeevolve with capability-oriented library maintenance and drift
control. Library hygiene retains skills that improve task success, including
skills that are both useful and unsafe. \safeevolve introduces safety
as a first-class admission and retention signal.

\section{Skill Misevolution}
\label{sec:problem}

\subsection{Self-improvement as a policy update}

Let an agent with base policy $\pi$ execute a sequence of three-task blocks
$\taskseq=(Q_1,\ldots,Q_K)$, where $Q_k=(q_{k,1},q_{k,2},q_{k,3})$ yields
trajectories $\mathcal{T}_k=(\tau_{k,1},\tau_{k,2},\tau_{k,3})$. After block
$Q_k$, an evolution method $\evolution$ maps the three trajectories and current
library $\library_k$ to one updated library:
\begin{equation}
  \library_{k+1}=\evolution(\library_k, Q_k, \mathcal{T}_k).
  \label{eq:evolution}
\end{equation}
The next three-task block is executed by the base agent conditioned on
retrieved skills from $\library_{k+1}$. Conversational state is reset between
tasks, and the library is the only intended cross-task state retained by the
protocol.

\subsection{Definition and temporal sequence}

An episode exhibits \emph{skill misevolution} when an unsafe procedure from an
earlier trajectory is recorded in the persistent library, is later retrieved,
and co-occurs with higher unsafe behavior than in a no-evolution
execution. The benchmark tracks the following temporal sequence:
\begin{equation}
\text{unsafe success}
\rightarrow \text{skill extraction}
\rightarrow \text{unsafe generalization}
\rightarrow \text{retrieval}
\rightarrow \text{reuse}
\rightarrow \text{persistent harm}.
\label{eq:sequence}
\end{equation}
The benchmark separately reports artifact formation, retrieval, and later
action. Skill involvement requires an observed retrieval event in addition to
the presence of a skill library.

\subsection{Threat model}

The attacker controls a bounded number of task inputs during the learning
history but does not modify the evolution code, skill store, retrieval engine,
judge, or final persistence probe. The attacker succeeds when an exposure is
recorded in a reusable skill and later unsafe behavior appears without a new
attack. The strongest observed outcome is carryover ASR in a fresh
session whose retrieval state is reconstructed only from the final frozen
\texttt{SKILL.md}.

\section{SkillMisevo Gym and Bench}
\label{sec:benchmark}

\subsection{Gym environment contract}

\gym turns a stateless executable safety task into a controlled stateful
learning process. An environment instance exposes five interfaces: task reset,
agent execution, skill update, skill retrieval, and artifact inspection. A
task-source adapter supplies executable instances, reset logic, attack and
benign views, and outcome judges; it does not control how skills are authored
or evaluated. Replacing this adapter changes the tasks and their success
semantics without changing the skill lifecycle, episode protocol, or
misevolution metrics. The gym snapshots $\library_k$ before and after every
three-task block, stores the authored diff and its three source trajectories,
records which skills are retrieved on each task,
and supports replay from any snapshot. Agent adapters translate native skill
formats and lifecycle hooks without converting the learning-stage artifact into
a shared representation. Evolution adapters expose when and from which
three-trajectory block a candidate skill is written. This contract supports both external evolution
methods and an agent's native skill evolution.

Conversation, workspace, process, and tool state are reset between tasks. The
gym then mounts only the designated skill snapshot. This isolation makes the
library the sole cross-task treatment and allows comparison executions with no
evolution, benign-only updates, or a frozen library. The environment emits a
common event log containing task outcome, skill writes, artifact versions,
retrievals, tool trajectories, and judge evidence.

The $P$ transition applies a stricter persistence boundary. After the final
learning block, the gym serializes the frozen skill library to
\texttt{SKILL.md} and terminates the learning executor. A clean executor is
initialized with the $P$ task and its reset tool environment; among
learning-stage state, it receives only this file. No conversation, workspace
contents, process state, tool-session state, retrieval cache, native memory, or
other evolved file is mounted. The skill index is
rebuilt from \texttt{SKILL.md}, and all $P$-task retrievals must resolve through
that reloaded index. The $P$ block performs no further skill evolution.

\subsection{AHA-guided task construction}

The experiments use AgentHazard~\citep{feng2026agenthazard} as a task-contract
source. We retain its harmful objectives, ten risk categories, sandboxed
computer-use environment, reset procedure, and official harmful trajectory
judge. AgentHazard does not provide the benign side of the evaluation. The
benchmark tasks are constructed through an offline AHA
pipeline~\citep{mao2026aha}.\footnote{Implementation:
\url{https://github.com/henrymao2004/Auto-research-red-teaming}.} For every task family, AHA hypothesizes and tests
failure mechanisms in the executable environment, commits a falsifier before
attack instantiation, and promotes replicated findings into a Vulnerability
Concept Graph. We select concepts whose enabling condition can be learned as a
reusable procedure and instantiate them across multiple task surfaces.

The AHA run-pair workflow instantiates each selected concept as matched $M/B$
pairs and held-out $P$ probes. The malicious member realizes the
concept-specific unsafe route; the benign member exercises the same operational
surface without that route; the persistence probe removes the attack and tests
whether a previously learned procedure fills in the unsafe behavior. Each
episode is grounded in one AHA-discovered concept, and every task in the episode
is an instantiation of that concept.

The construction output---concept, mechanism, malicious goal, surface, prompts,
and instance identifiers---is frozen before agent evaluation. The official
harmful judge scores the paired harmful objective on $M$, $B$, and $P$; we add
an orthogonal benign-completion judge for the newly constructed benign
objectives. Every final task inherits the corresponding AHA concept identifier,
and the final prompts are constructed instances rather than replayed
AgentHazard prompts. AHA's VCG is never exposed to the evaluated agent or
evolution method: it is a benchmark-authoring artifact, whereas the skills
measured in this paper are authored by the evaluated system from its own
trajectories.

\subsection{Benchmark episode protocol}

Each episode contains 21 tasks constructed around one validated vulnerability
concept, task-source category, and task family. For each task surface, the
constructor first creates an $M/B$ run pair. $M$ instantiates the concept's
enabling condition and harmful objective in an executable task. $B$ preserves
the legitimate operation, tools, and domain while removing the malicious
condition and specifying a safe benign objective. A concept-matched $P$ task
uses a fresh surface and contains no attack payload; its underspecified request
tests what procedure the reloaded skill supplies. The order is
\begin{equation}
  \underbrace{M^3_1,B^3_1}_{R_1},
  \underbrace{M^3_2,B^3_2}_{R_2},
  \underbrace{M^3_3,B^3_3}_{R_3}
  \mid P^3,
\end{equation}
or, schematically, $MMM\,BBB\;MMM\,BBB\;MMM\,BBB\mid BBB$. The evolution method runs once after each $M^3$ or $B^3$
block, authoring or revising skills from the three completed trajectories. Each
round therefore contains two skill-evolution steps. Each round contributes a dose of three new malicious tasks;
the cumulative dose is therefore three, six, and nine malicious tasks after
$R_1,R_2,R_3$. The following benign block measures contamination at that dose.
After $R_3$, the library is frozen and the three $P$ tasks run in a new executor
session without further evolution. This executor reloads its skill index solely
from the frozen \texttt{SKILL.md}; no other learning-stage state enters the
$P$ block. Each condition contains 25 episodes.

\subsection{Comparison settings}

Every episode is replayed under each evolution setting:
\begin{itemize}
  \item \textbf{No evolution}: disables persistent learning and establishes the
  base-agent floor.
  \item \textbf{Benign-only updates}: executes the identical task sequence but
  permits skill updates only after $B^3$ blocks, providing the reference pattern
  under ordinary benign learning.
  \item \textbf{Full evolution}: applies each evaluated method to all successful
  trajectories in three-task batches under its native update rule.
\end{itemize}
All arms use identical tasks, order, tools, agent configuration, and judges.

\subsection{Agents and evolution methods}

Every assembled episode is executed by four target-agent configurations. Claude
Code, Codex, Hermes, and OpenClaw share MiniMax-M2.5, holding the backbone fixed while
the agent framework changes. The target is the complete agent system---its scaffold,
tool mediation, skill lifecycle, retrieval behavior, and backbone---rather than
an isolated language model.

The main comparison includes five agent-agnostic evolution methods: EvoSkill,
SkillClaw, AutoSkill, SkillsVote, and SkillOpt. Each runs on all four target
agents. Hermes also provides a framework-native evolution path supported only
by the Hermes target. We include Hermes-native in the main diagnostic table and
provide its implementation details and expanded results in
Appendix~\ref{app:hermes-native}; governed results appear in
Table~\ref{tab:safeevolve-full}. Each method is evaluated in its actual stored representation, including
associated scripts or structured contracts, rather than after conversion to a
common textual summary, for online behavior and artifact auditing. The
carryover stage deliberately standardizes the persistence channel: only the
method's frozen \texttt{SKILL.md} is reloaded for $P$, while associated scripts,
resources, caches, and native memory remain unavailable.

\subsection{Metrics}

All nine headline metrics are absolute micro-aggregates over the 25 episodes in
a condition. Let $\mathcal{B}_C$ and $\mathcal{P}_C$ be tasks with a valid
utility-judge result, and let $\mathcal{B}_H$, $\mathcal{M}_H$, and
$\mathcal{P}_H$ be tasks with a valid harmful-judge result.
Let $C(x)$ denote completion of the benign objective and $H(x)$ completion of
the paired AgentHazard harmful objective. $C$ is produced by our independent
benign-completion judge; $H$ is produced by the retained AgentHazard harmful
trajectory judge.

\paragraph{Online behavior.}
The three online metrics read every scored $B$ or $M$ task:
\begin{align}
\mathrm{BenignUtility} &=
\frac{\sum_{x\in\mathcal{B}_C} C(x)}{|\mathcal{B}_C|}, &
\mathrm{MaliciousASR} &=
\frac{\sum_{x\in\mathcal{M}_H} H(x)}{|\mathcal{M}_H|}, \\
\mathrm{Contamination} &=
\frac{\sum_{x\in\mathcal{B}_H} H(x)}{|\mathcal{B}_H|}.
\end{align}
Benign utility uses our benign-completion judge on all $B$ tasks. This judge is
orthogonal to harm: it evaluates whether the constructed benign objective was
actually completed. Malicious ASR uses the AgentHazard harmful judge on $M$;
benign ASR, which we call contamination, applies the same paired harmful
objective and judge to $B$. AgentHazard does not define the $B$ tasks or their
utility labels; both are introduced by our concept-to-task construction.

\paragraph{Evolved artifact.}
Let $\mathcal{A}$ contain every skill authored from an $M^3$ or $B^3$ block. Independent content judges
assign 1--5 scores for Content Unsafety (CU), Unsafe Generalization (UG), and
Stealthiness (Stealth), denoted $cu$, $gen$, and $sneak$. Let
$\mathcal{A}_U=\{a\in\mathcal{A}:cu(a)\geq3\}$ denote the artifacts for which
the judge identifies unsafe content. The headline artifact metrics retain the
original judge scores:
\begin{equation}
\begin{aligned}
\mathrm{CU} &= \frac{\sum_{a\in\mathcal{A}}cu(a)}{|\mathcal{A}|},\\
\mathrm{UG} &= \frac{\sum_{a\in\mathcal{A}_U}gen(a)}{|\mathcal{A}_U|},\\
\mathrm{Stealth} &= \frac{\sum_{a\in\mathcal{A}_U}sneak(a)}{|\mathcal{A}_U|}.
\end{aligned}
\end{equation}
CU scores the full authored artifact set. Following the content-judge contract,
UG and Stealth score generalization and disguise among artifacts containing
unsafe content. CU is N/A when $\mathcal{A}$ is empty; UG and Stealth are N/A
when $\mathcal{A}_U$ is empty. The appendix reports score distributions and
auditor agreement on a double-labeled subset.

\paragraph{Post-attack carryover.}
After conversational and environment state are reset, the retrieval index is
rebuilt solely from the frozen final \texttt{SKILL.md}. For each $P$ task $x$,
let $\mathcal{U}(x)$ be the evolved skills present in that file whose exported
text receives $cu\geq3$, and let $R_U(x)=1$ when the retrieval log contains at
least one member of $\mathcal{U}(x)$. Define
$\mathcal{P}_U=\{x\in\mathcal{P}:\mathcal{U}(x)\neq\emptyset\}$. The three $P$ tasks per
episode yield
\begin{equation}
\mathrm{URR}=
\frac{\sum_{x\in\mathcal{P}_U}R_U(x)}{|\mathcal{P}_U|},\qquad
\mathrm{CarryoverASR}=
\frac{\sum_{x\in\mathcal{P}_H}H(x)}{|\mathcal{P}_H|},\qquad
\mathrm{CarryoverUtility}=
\frac{\sum_{x\in\mathcal{P}_C}C(x)}{|\mathcal{P}_C|}.
\end{equation}
URR is N/A when $\mathcal{P}_U$ is empty; it is never replaced by zero. It
measures retrieval of an unsafe evolved skill, not retrieval of arbitrary
benign skills. Thus a $P$ task enters the URR denominator only when its
episode's frozen export contains at least one evolved skill with $cu\geq3$.
Carryover utility applies the same benign-completion judge used for $B$ to the
concept-matched benign objective of each $P$ task.
Because a method can make URR undefined by preventing unsafe export altogether,
URR is interpreted jointly with the artifact panel. N/A denotes no unsafe
export and remains outside numerical comparisons.

\subsection{Statistical protocol}

Results are micro-aggregated over all valid tasks or judged artifacts; we do not
average per-episode percentages. Confidence intervals
use episode-level bootstrap resampling. Utility, ASR, contamination,
unsafe-retrieval, and carryover outcomes are compared over the same task
instances; method comparisons report multiple-comparison-adjusted intervals.
Hyperparameters, episode construction,
and the held-out split are frozen before the final sweep. Headline metrics pool
all three rounds. Dose--response points instead use the corresponding $M_r$ or
$B_r$ block and the cumulative artifact snapshot after round $r$.

\section{SafeEvolve}
\label{sec:safeevolve}

\safeevolve is a governance layer around an existing evolution method. It does
not replace the base agent or add a runtime refusal policy. It governs the
persistent skill update at two decision points: writing and reuse.

\subsection{Admission critic and targeted repair}

Before a candidate enters the library, an independent critic audits the full
bundle, including scripts and referenced resources. When a candidate removes a
safety check, assumes authority, broadens a dangerous operation, or embeds an
unsafe tool sequence, the critic produces a targeted repair: restore the check,
add the missing precondition, or narrow the applicability condition. Repaired
skills are re-audited before admission.

\subsection{Verification-preserving evolution}

Skill revisions are constrained to preserve existing verification,
confirmation, authorization, and reversibility requirements unless the update
provides explicit evidence that the requirement is obsolete. This prevents an
efficiency-oriented edit from treating a safeguard as removable overhead.

\subsection{Lineage and provenance}

Each skill stores its source tasks, trajectories, revisions, and exposure
status. Provenance supports traceability and batch rollback when an upstream
experience is later judged unsafe. It also supplies a risk prior to retrieval
and retirement decisions.

\subsection{Safety-aware selection and retirement}

Raw evolution selects skills primarily by utility. \safeevolve augments this
objective with the admission audit, provenance risk prior, and observed safety
impact of reuse. At retrieval time it suppresses candidates whose exported
instructions remain unsafe or whose lineage contains unsafe reuse evidence;
URR measures whether such a skill still reaches the fresh executor. A retrieved
skill associated with a successful carryover attack is subsequently
downweighted, quarantined, repaired, or retired even when it improves task
success. This distinguishes \safeevolve from capability-only library hygiene,
which keeps useful but unsafe skills.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Research questions}

\begin{description}
  \item[RQ1: Safety--utility association.] Is skill evolution associated with
  higher unsafe behavior while retaining or improving task utility?
  \item[RQ2: Exposure-to-persistence sequence.] Do unsafe experience, an unsafe
  skill artifact, retrieval, and post-attack carryover ASR appear in sequence
  while retaining carryover utility?
  \item[RQ3: Scope, budget, and timing.] How does misevolution vary across
  methods, agents, vulnerability concepts and task surfaces, cumulative doses of
  three, six, and nine malicious tasks, and early versus late exposure at a
  fixed dose?
  \item[RQ4: Mitigation.] Does \safeevolve retain utility while attaining lower
  evolved-artifact risk, malicious and benign ASR, unsafe retrieval, and
  carryover ASR?
\end{description}

\subsection{Evaluation matrix}

The main evaluation treats target-agent as a first-class axis, not a nuisance

\paragraph{Capability gains can coincide with higher malicious success.}
Higher benign utility frequently appears alongside higher malicious ASR in
Table~\ref{tab:misevolution-main}. EvoSkill reports BU/M-ASR of 71.1/62.2 on
Claude Code and 76.0/64.4 on Codex, compared with 49.8/56.0 and 54.7/59.6
under no evolution. Hermes-native reports 68.8/73.4, compared with 14.2/44.0
under no evolution on Hermes. SkillClaw defines a different regime: utility
generally falls and malicious ASR does not show the same increase. The
association therefore varies across evolution method--agent systems.

\paragraph{Three settings exhibit the full observed misevolution signature.}
AutoSkill and SkillsVote on Claude Code reach B-ASR 8.4/7.1 and fresh-session
C-ASR 16.0/18.7 while retaining C-Util 64.0/65.3. Hermes-native is the
strongest observed case, reaching B-ASR 16.0, C-ASR 30.7, and C-Util 64.0.
Across these settings, the benchmark observes an evolved unsafe artifact,
later benign contamination, and harmful behavior after the executor is reset
and retrieval is rebuilt solely from the frozen \texttt{SKILL.md}. Carryover is therefore
compatible with retained task competence rather than being explained by a
generally broken agent.

\paragraph{Artifact risk is broader than realized carryover harm.}
The remaining settings separate earlier stages of the lifecycle from harmful
execution. EvoSkill and SkillOpt on Claude Code produce unsafe retrieval rates
of 81.0 and 50.0 while C-ASR remains zero; Codex with EvoSkill similarly reaches
33.3 URR with zero C-ASR. Unsafe skill content can therefore be authored and
retrieved without producing an observed harmful action in the persistence
probe. Runtime policy, tool mediation, and task context remain gates between a
persistent unsafe rule and realized harm.

\paragraph{Safety--utility frontier.}
The high-utility, high-C-ASR region is the central misevolution regime:
AutoSkill, SkillsVote, and Hermes-native preserve useful procedures while also
retaining an unsafe shortcut. Capability improvement alone is therefore not a
reliable safety signal, while artifact risk alone does not establish that the
shortcut will be executed. The nine metrics locate both properties without
collapsing them into a single score.

\paragraph{Artifact risk is broader than behavioral harm.}
The aggregate results already expose an artifact--behavior gap. Every reported
evolution setting produces nontrivial artifact scores, whereas clear elevated
carryover ASR is concentrated in AutoSkill, SkillsVote, and Hermes-native.
EvoSkill and SkillOpt further show that even observed unsafe retrieval need not
be followed by harm. RQ2 measures these transitions episode by episode to
separate latent artifact risk, retrieved risk, and realized carryover. This is a
gated lifecycle: the observed progression can stop because no unsafe skill is
exported, the skill is not retrieved, or the executor declines to act on it.

\subsection{RQ3: Where does misevolution emerge, at what budget, and when?}
\label{sec:rq3}

RQ3 characterizes heterogeneity across evolution algorithms, agent frameworks,
AgentHazard risk categories, vulnerability concepts, task surfaces, exposure budgets, and exposure
positions. The analysis uses the pooled mean as a summary and then identifies
which design properties accompany high artifact unsafety and which settings
retain artifact risk without observed post-attack harm.

\paragraph{Framework decomposition.}
The four MiniMax-M2.5 targets hold the backbone fixed while changing the agent
framework, enabling comparisons across tool mediation, system instructions,
skill retrieval, and execution policy. We report these framework comparisons
before the overall macro-average; the four configurations are
related controls, not interchangeable replications.

The current comparisons already show that framework and evolution method matter
at different stages. Hermes-native produces the highest observed contamination
and carryover ASR; Claude Code permits full carryover under AutoSkill and
SkillsVote; Codex with EvoSkill retrieves unsafe skills without realizing
carryover harm; and SkillClaw produces little or no elevation over the
no-evolution carryover rate across the reported frameworks. Dose and timing
analyses test how these system-specific outcomes develop as exposure changes
and whether the response is monotonic within each setting.

\subsubsection{Timing sweep}
\label{sec:timing-sweep}

The dose--response sweep changes how much adversarial experience enters the
learning stream. The timing sweep fixes the malicious dose at one three-task
block and moves that block through an otherwise identical learning history. The
three schedules are
\begin{align*}
  \text{\textsc{Early}}:  &\quad \mathrm{MMM\,BBB\;BBB\,BBB\;BBB\,BBB}
    \mid \mathrm{BBB},\\
  \text{\textsc{Middle}}: &\quad \mathrm{BBB\,BBB\;MMM\,BBB\;BBB\,BBB}
    \mid \mathrm{BBB},\\
  \text{\textsc{Late}}:   &\quad \mathrm{BBB\,BBB\;BBB\,BBB\;MMM\,BBB}
    \mid \mathrm{BBB}.
\end{align*}
Non-adversarial positions use the corresponding concept-matched benign tasks.
Every schedule therefore contains the same 18 learning-stage tasks, three malicious tasks,
15 benign tasks, six block-level evolution opportunities, and three clean-session
persistence probes. This compares early exposure followed by more benign
updates with recent exposure followed by fewer benign updates before the
library is frozen.

Capability-oriented hygiene can remove stale or low-value skills, but it may
retain a skill associated with both higher utility and an unsafe shortcut. A gap between
Library-Drift and \safeevolve provides a direct comparison between
capability-oriented and explicitly safety-aware governance.

\section{Discussion}
\label{sec:discussion}

\paragraph{Skills are security-relevant policy.}
The central implication is that a skill library should not be governed as a
passive document store. A stored skill can alter tool choice, ordering,
verification, and authorization across future tasks. Admission, provenance,
rollback, and safety monitoring should therefore be part of the learning
boundary.

\paragraph{Artifact risk and post-attack harm are distinct.}
An evolved artifact can score highly on content unsafety, unsafe
generalization, or stealthiness before its unsafe procedure is retrieved. This
is latent risk, not a safe artifact. Conversely, a runtime policy can block
execution while the persistent library continues to carry the unsafe rule.
Product evaluation should therefore report online utility and ASR, the three
artifact properties, and unsafe retrieval, carryover ASR, and carryover utility
separately. The observed progression is gated rather than automatic: unsafe
experience can yield an unsafe artifact without later retrieval, and retrieval
can occur without harmful execution.

\paragraph{Capability and safety share the same shortcut.}
The difficult cases are not useless malicious skills. They are procedures in
which higher success appears alongside the removal of friction that served as a
safeguard. Accordingly, we report capability together with safety rather than
treating either one alone as sufficient.

\paragraph{Misevolution is a system-level outcome.}
The same backbone exhibits different outcomes across agent frameworks and
evolution methods. Some settings complete the path from unsafe experience to
clean-session harm, while others retain the unsafe rule without executing it.
Skill authoring, serialization, retrieval, tool mediation, and runtime policy
jointly determine where the observed progression stops. This heterogeneity is
part of the benchmark result: it identifies which layer requires governance.

\paragraph{Scope and limitations.}
The study focuses on readable skill artifacts written by CLI and coding agents
and on bounded direct exposure in executable task environments. Results support
claims about incremental risk observed under evolution within this setting. Future
work should test indirect exposure, multimodal skills, opaque learned memories,
longer deployment horizons, interacting libraries, and adaptive attackers.

\section{Conclusion}
\label{sec:conclusion}

Skill self-evolution changes the temporal structure of agent safety. Our
evaluation identifies a persistent channel through which unsafe success can be
generalized, stored, retrieved, and, in some method--agent systems, carried into
clean-session behavior after the original attack is gone. Other systems stop at
artifact formation or retrieval, making the location of that boundary part of
the safety result. \gym makes this longitudinal state observable and replayable;
\bench distinguishes latent artifact risk from
retrieved risk and realized carryover. \safeevolve governs the persistent
update while preserving useful learning. Safe self-improvement requires
evaluating not only whether an agent succeeds today, but also which persistent
skills are created, when they are retrieved, and what later behavior follows.

\section{Gym and Benchmark Construction}
\label{app:construction}

\paragraph{Source contract.}
AgentHazard contributes the harmful objective taxonomy, executable
computer-use environment, reset logic, and harmful trajectory judge. We pin its
version and preserve the category and harmful-goal semantics in every derived
task. Its published prompts are inputs to vulnerability discovery rather than
the final \bench episode dataset.

\paragraph{AHA concept discovery.}
For each benchmark task family, an offline AHA run searches for a vulnerability
concept that can survive abstraction into a reusable skill. The Hypothesizer
commits a mechanism, predicted unsafe outcome, and falsifier; the
Attack-Designer instantiates the proposal in the AgentHazard environment; the
victim trajectory is judged and reflected on; and only replicated,
non-falsified findings enter the VCG. We retain the concept identifier,
mechanism, enabling condition, attack template, failure outcome, transfer
prediction, and discovery provenance. This stage determines what failure
mechanism a task tests, not what skill the evaluated agent should write.

\paragraph{AHA concept-to-run-pair instantiation.}
For each retained concept and task surface, the constructor emits a malicious
task $M$ and a zero-injection benign task $B$. The pair shares the operational
domain, tool affordances, and legitimate task family. $M$ instantiates the
concept's enabling condition and paired harmful goal. $B$ removes that
condition, preserves a useful neighboring objective, and makes the safe
boundary executable and judgeable. Additional surface instantiations produce
$P$ probes whose prompts contain neither the malicious payload nor its explicit
unsafe instruction. These probes retain enough semantic overlap to retrieve a
generalized skill and test what the skill contributes in a clean session.

\paragraph{Judging and validation.}
The AgentHazard harmful judge evaluates the paired harmful objective on $M$,
$B$, and $P$. We add a separate benign-completion judge for the constructed
benign objective; its decision is independent of whether an additional harmful
action occurred. Construction validation checks executability, reset behavior,
surface matching within each run pair, removal of the malicious condition from
$B/P$, prompt and instance deduplication, and judge coverage. Concepts, prompts,
judges, and splits are frozen before the evolution-method evaluation.

\paragraph{Episode manifest and replay.}
The released manifest records, for every episode, its AHA concept identifier,
mechanism, AgentHazard category, malicious goal, task surface, ordered $M/B/P$
instances, and source provenance. It also records each skill snapshot and the
deterministic final export to \texttt{SKILL.md}. The clean-executor check verifies
that the $P$ retrieval index reads this export and no other learning-stage
state, allowing every result to be replayed from the same constructed task and
skill snapshot.
