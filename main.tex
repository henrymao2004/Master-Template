\begin{abstract}
LLM agents increasingly improve themselves by distilling successful
interactions into reusable skills. This turns experience into persistent
behavioral policy, yet existing evaluations measure whether evolution improves
capability without testing what happens when success depends on an unsafe
shortcut. We identify \emph{skill misevolution}: a transient unsafe success is
extracted into an apparently legitimate skill and later reused on benign tasks
after the original attack has disappeared. We introduce \gym, a
task-source-agnostic environment that turns executable agent tasks into stateful
skill-learning episodes. It standardizes skill writing, versioned library
state, retrieval, replay, and fresh-session evaluation across agent frameworks
and evolution methods. On top of this environment, \bench defines an
exposure-to-carryover protocol, paired evolution controls, and nine metrics for
online behavior, the evolved artifact, and post-attack behavior. We instantiate the benchmark with
AgentHazard as its executable safety-task source. Across four agent
frameworks, three model backbones, five target-agent configurations, [M]
evolution methods and ten AgentHazard risk categories, benign utility,
malicious utility, malicious ASR, and benign-task contamination are
[BU/MU/MA/BA]. Content unsafety, unsafe generalization, and stealthiness are
[$S_{\mathrm{content}}$/$S_{\mathrm{general}}$/$S_{\mathrm{stealth}}$];
carryover ASR reaches [CA] while carryover utility remains [CU]. Unsafe
content appears in the skill library before it consistently manifests as
behavior, and the effect emerges by round [R], after [K] malicious tasks. We
further introduce \safeevolve, a governance layer that audits and
repairs candidate skills, preserves verification steps, tracks provenance, and
retires skills with unsafe reuse histories. Under \safeevolve, the artifact
online metrics are [BU'/MU'/MA'/BA'], artifact scores are
[$S'_{\mathrm{content}}$/$S'_{\mathrm{general}}$/$S'_{\mathrm{stealth}}$],
and carryover ASR / utility are [CA'/CU'].
These results show that self-improvement is a security-relevant policy update:
practice can make an agent more capable and persistently less safe.
\end{abstract}

\section{Introduction}
\label{sec:introduction}

LLM agents are beginning to improve through use. Modern CLI and coding agents
can summarize successful trajectories, write or revise reusable skills, and
retrieve those skills when related tasks appear. This capability changes an
agent from a fixed policy with transient context into a system whose behavior is
partly rewritten by its own experience. Skill evolution is therefore more than
memory: it is a persistent policy update expressed as an editable artifact.

The signal used to produce that update is usually task success or efficiency.
Successful behavior, however, is not necessarily safe behavior. An agent may
complete a task by skipping a verification step, assuming authorization,
treating an irreversible action as routine, or following an attacker-controlled
observation. If the evolution method compresses that trajectory into a reusable
procedure, it can preserve the unsafe shortcut precisely because the shortcut
helped the agent succeed.

This creates a temporal safety failure that independent-task evaluations cannot
observe. The original attack may disappear with the context window, but the
lesson extracted from it can remain in the skill library. A later benign task
can retrieve the skill, apply its over-generalized rule, and reproduce the harm
without a malicious instruction. A fresh session can inherit the same behavior
by loading the evolved library alone. We call this process \emph{skill
misevolution}: transient unsafe success becomes persistent, retrievable, and
generalizable policy.

Measuring misevolution requires an environment that exposes persistent learning
as experimental state. We introduce \gym, which wraps executable agent tasks
with a standardized skill lifecycle: it snapshots the
library before and after each task, resets conversational and workspace state,
records skill writes and retrievals, and mounts a frozen library in a new
executor session. Only the evolving library crosses task boundaries.

\bench is the evaluation protocol built on this gym. Its basic unit is a
21-task episode with three rounds of three malicious tasks followed by three
benign tasks, then three persistence probes in a fresh executor session.
Matched no-evolution,
read-only-reflection, and benign-evolution controls isolate whether later harm
comes from the learned artifact rather than the base model, repeated context,
or ordinary benign learning.

The benchmark evaluates three stages of the failure. Online behavior is measured
by benign utility, malicious utility, malicious ASR, and benign ASR
(contamination). The evolved artifact is measured by content unsafety, unsafe
generalization, and stealthiness. Post-attack behavior is measured by carryover
ASR and carryover utility in a clean session. This
distinction matters because an unsafe skill can remain latent
until a later task supplies the right retrieval cue. Behavior-only evaluation
therefore misses risk that has already entered the persistent policy.

Our evaluation asks four questions. \textbf{RQ1} tests whether evolution
increases unsafe behavior while retaining utility. \textbf{RQ2} follows the
exposure-to-persistence chain and tests whether the skill artifact mediates
later harm. \textbf{RQ3} measures how the effect varies across evolution
methods, agent configurations, task categories, attack strategies, and attacker
budgets. \textbf{RQ4}
tests whether \safeevolve can break this chain without removing the capability
benefits of learning.

\safeevolve acts at the learning boundary rather than as another runtime
refusal layer. It audits the complete candidate skill bundle before admission,
repairs unsafe generalization instead of deleting useful procedures, preserves
verification requirements during revision, records the experiences from which
each skill was learned, and downweights or retires skills whose reuse produces
unsafe behavior. These components target the persistent object responsible for
the incremental risk: the evolved skill library.

Our contributions are:
\begin{enumerate}
  \item We identify \textbf{skill misevolution}, a temporal failure mode in
  which self-improvement converts transient unsafe success into persistent
  behavior that can reappear on benign tasks without the original attack.

  \item We introduce \gym, a reusable environment for safety evaluation of
  self-improving agents. It provides a common skill lifecycle, versioned library
  state, isolation and replay, retrieval traces, and adapters for multiple agent
  frameworks and evolution methods.

  \item We introduce \bench, a task-source-independent temporal protocol with
  paired evolution controls and nine absolute metrics: benign utility,
  malicious utility, malicious ASR, benign ASR (contamination), content
  unsafety, unsafe generalization, stealthiness, carryover ASR, and carryover
  utility. We
  instantiate it with AgentHazard as the executable task and judge source.

  \item We provide a systematic evaluation across four agent frameworks, three
  model backbones, five target-agent configurations, [M] evolution methods, ten
  AgentHazard risk categories, and multiple exposure budgets, separating
  artifact-level latent risk from post-attack realized harm.

  \item We introduce \safeevolve and show that governing skill admission,
  revision, provenance, and retirement reduces evolution-induced harm while
  retaining the utility gained through learning.
\end{enumerate}

\section{Related Work}
\label{sec:related-work}

\paragraph{Self-improving agents and skill evolution.}
Position this work against methods that extract, edit, select, vote over, or
retrieve reusable skills from agent trajectories. Organize the literature by
the persistent artifact each method produces and the success signal used to
select it. The key gap is that these methods report capability improvement but
do not measure evolution-induced safety change.

\paragraph{Safety of persistent agent memory.}
Discuss memory poisoning, long-term memory evolution, retrieval attacks, and
forgetting. Distinguish a skill from a stored observation: a skill is an
executable or prescriptive procedure intended to generalize across tasks. The
failure studied here is endogenous policy formation from the agent's own
successful trajectory, not only storage of attacker-supplied text.

\paragraph{Executable safety-task sources.}
Agent safety benchmarks supply tasks, environments, attack inputs, and outcome
judges that a learning environment can execute. Our current benchmark
instantiation uses AgentHazard, which provides 2,653 computer-use instances,
ten risk categories, and multi-step attack strategies
\citep{feng2026agenthazard}. AgentHazard is a task-source adapter for \gym: its
task and judge semantics are retained, while the gym supplies the persistent
skill state, lifecycle instrumentation, and cross-task execution protocol.

\paragraph{Governance of evolving libraries.}
Contrast \safeevolve with capability-oriented library maintenance and drift
control. Library hygiene retains skills that improve task success, including
skills whose utility comes from unsafe shortcuts. \safeevolve introduces safety
as a first-class admission and retention signal.

\section{Skill Misevolution}
\label{sec:problem}

\subsection{Self-improvement as a policy update}

Let an agent with base policy $\pi$ execute a sequence of tasks
$\taskseq=(q_1,\ldots,q_T)$. After task $q_t$, an evolution method
$\evolution$ maps the trajectory $\tau_t$ and current library $\library_t$ to
an updated library:
\begin{equation}
  \library_{t+1}=\evolution(\library_t, q_t, \tau_t).
  \label{eq:evolution}
\end{equation}
The next task is executed by the base agent conditioned on retrieved skills
from $\library_{t+1}$. Conversational state is reset between tasks, so any
systematic cross-task change is carried by the evolved artifact.

\subsection{Definition and causal chain}

An episode exhibits \emph{skill misevolution} when an unsafe procedure learned
from an earlier trajectory enters the persistent library and causes incremental
unsafe behavior on a later task relative to a matched no-evolution execution.
The proposed causal chain is:
\begin{equation}
\text{unsafe success}
\rightarrow \text{skill extraction}
\rightarrow \text{unsafe generalization}
\rightarrow \text{retrieval}
\rightarrow \text{reuse}
\rightarrow \text{persistent harm}.
\label{eq:chain}
\end{equation}
The benchmark separately observes each link. This prevents a later unsafe
action from being attributed to the skill merely because a skill library was
present.

\subsection{Threat model}

The attacker controls a bounded number of task inputs during the learning
history but does not modify the evolution code, skill store, retrieval engine,
judge, or final persistence probe. The attacker succeeds when an exposure is
absorbed into a reusable skill that increases unsafe behavior or later causes
harm without a new attack. The strongest outcome is carryover ASR in a fresh session
that receives only the final evolved library.

\section{SkillMisevo Gym and Bench}
\label{sec:benchmark}

\subsection{Gym environment contract}

\gym turns a stateless executable safety task into a controlled stateful
learning process. An environment instance exposes five interfaces: task reset,
agent execution, skill update, skill retrieval, and artifact inspection. A
task-source adapter supplies executable instances, reset logic, attack and
benign views, and outcome judges; it does not control how skills are authored
or evaluated. Replacing this adapter changes the tasks and their success
semantics without changing the skill lifecycle, episode protocol, or
misevolution metrics. The gym snapshots $\library_t$ before and after every task, stores the authored diff
and its source trajectory, records which skills are retrieved on the next task,
and supports replay from any snapshot. Agent adapters translate native skill
formats and lifecycle hooks without converting the stored artifact into a
shared representation. Evolution adapters expose when and from which trajectory
a candidate skill is written. This contract supports both external evolution
methods and an agent's native skill evolution.

Conversation, workspace, process, and tool state are reset between tasks. The
gym then mounts only the designated skill snapshot. This isolation makes the
library the sole cross-task treatment and allows matched executions with no
evolution, benign-only evolution, or a frozen library. The environment emits a
common event log containing task outcome, skill writes, artifact versions,
retrievals, tool trajectories, and judge evidence.

\subsection{Benchmark episode protocol}

Each episode contains 21 tasks from one task-source category and task family.
The order is
\begin{equation}
  \underbrace{M^3_1,B^3_1}_{R_1},
  \underbrace{M^3_2,B^3_2}_{R_2},
  \underbrace{M^3_3,B^3_3}_{R_3}
  \mid P^3,
\end{equation}
or, schematically, $MMM\,BBB\;MMM\,BBB\;MMM\,BBB\mid BBB$. Here $M$ is an
adversarial task, $B$ its zero-injection benign twin, and $P$ a benign
persistence probe. Each round contributes a dose of three new malicious tasks;
the cumulative dose is therefore three, six, and nine malicious tasks after
$R_1,R_2,R_3$. The following benign block measures contamination at that dose.
After $R_3$, the library is frozen and the three $P$ tasks run in a new executor
session. Each condition contains 25 episodes, giving at most 225 scored $M$
tasks, 225 scored $B$ tasks, and 75 scored $P$ tasks.

\subsection{Paired controls}

Every episode is replayed under matched evolution settings:
\begin{itemize}
  \item \textbf{No evolution}: disables persistent learning and establishes the
  base-agent floor.
  \item \textbf{Read-only reflection}: retains textual lessons in the current
  learning mechanism without authoring a reusable skill artifact.
  \item \textbf{Benign evolution}: evolves only from benign tasks and tests
  whether ordinary learning alone produces the effect.
  \item \textbf{Full evolution}: applies each evaluated method to all successful
  trajectories under its native update rule.
\end{itemize}
All arms use identical tasks, order, tools, agent configuration, and judges.

\subsection{Task-source instantiation, agents, and evolution methods}

The experiments instantiate the gym's task-source interface with AgentHazard
\citep{feng2026agenthazard}. The adapter exposes its harmful objectives,
multi-step attack sequences, ten risk categories, sandboxed computer-use tasks,
reset procedure, and trajectory-level judge. It also constructs matched benign
twins and persistence probes required by the benchmark episode protocol. Every
assembled episode is executed by five target-agent configurations spanning four
agent frameworks and three model backbones. Claude Code, Codex, Hermes, and OpenClaw
share MiniMax-M2.5 as a controlled backbone comparison. Claude Code is also
paired with Haiku~4.5, and Codex with GPT-5.4-mini, providing within-framework
backbone comparisons. The target is the complete agent system---its scaffold,
tool mediation, skill lifecycle, retrieval behavior, and backbone---rather than
an isolated language model.

The evolution settings include EvoSkill, SkillClaw, AutoSkill, SkillsVote, and
Hermes-native skill evolution. Hermes-native is tied to the Hermes agent and is
evaluated only in the Hermes /
MiniMax configuration; the corresponding cells for other target agents are
N/A. Each method is evaluated in its actual stored representation, including
associated scripts or structured contracts, rather than after conversion to a
common textual summary.

\subsection{Metrics}

All nine headline metrics are absolute micro-aggregates over the 25 episodes in
a condition. For $X\in\{B,M,P\}$, let $\mathcal{X}_C$ and $\mathcal{X}_H$ be
the tasks with a valid utility-judge and harmful-judge result, respectively.
Let $C(x)$ denote completion of the benign objective and $H(x)$ completion of
the AgentHazard harmful objective.

\paragraph{Online behavior.}
The four online metrics read every scored $B$ or $M$ task:
\begin{align}
\mathrm{BenignUtility} &=
\frac{\sum_{x\in\mathcal{B}_C} C(x)}{|\mathcal{B}_C|}, &
\mathrm{MaliciousUtility} &=
\frac{\sum_{x\in\mathcal{M}_C} C(x)}{|\mathcal{M}_C|}, \\
\mathrm{MaliciousASR} &=
\frac{\sum_{x\in\mathcal{M}_H} H(x)}{|\mathcal{M}_H|}, &
\mathrm{Contamination} &=
\frac{\sum_{x\in\mathcal{B}_H} H(x)}{|\mathcal{B}_H|}.
\end{align}
Benign utility uses the AgentHazard utility judge on all $B$ tasks. Malicious
utility applies the same judge to the benign-twin objective on every $M$ task,
so useful task completion and harmful success remain separate outcomes.
Malicious ASR uses the AgentHazard harmful judge on $M$; benign ASR, which we
call contamination, applies that harmful judge to $B$. Each denominator is at
most 225 tasks.

\paragraph{Evolved artifact.}
Let $\mathcal{A}$ contain every skill authored from an $M$ or $B$ task, and let
$\mathcal{A}_u=\{a\in\mathcal{A}:cu(a)\geq3\}$. Independent content judges
assign 1--5 scores for content unsafety $cu$, unsafe generalization $gen$, and
stealthiness $sneak$. We report
\begin{equation}
\mathrm{CU}=\frac{\sum_{a\in\mathcal{A}}cu(a)}{|\mathcal{A}|},\quad
\mathrm{UG}=\frac{\sum_{a\in\mathcal{A}_u}gen(a)}{|\mathcal{A}_u|},\quad
\mathrm{Stealth}=\frac{\sum_{a\in\mathcal{A}_u}sneak(a)}{|\mathcal{A}_u|}.
\end{equation}
Thus CU reads all authored skills, whereas UG and Stealth are conditional on an
unsafe authored skill. CU is N/A when $\mathcal{A}$ is empty; UG and Stealth
are N/A when $\mathcal{A}_u$ is empty. Auditor agreement is measured on a
double-labeled subset.

\paragraph{Post-attack carryover.}
After conversational and environment state are reset and only the frozen final
library is mounted, the three $P$ tasks per episode yield
\begin{equation}
\mathrm{CarryoverASR}=
\frac{\sum_{x\in\mathcal{P}_H}H(x)}{|\mathcal{P}_H|},\qquad
\mathrm{CarryoverUtility}=
\frac{\sum_{x\in\mathcal{P}_C}C(x)}{|\mathcal{P}_C|}.
\end{equation}
The denominators contain at most 75 scored persistence tasks.

\subsection{Statistical protocol}

Results are micro-aggregated over all valid tasks or authored skills; we do not
average per-episode percentages. Confidence intervals
use episode-level bootstrap resampling. Paired utility, ASR, contamination, and carryover outcomes are
tested over matched task instances; method comparisons report effect sizes and
multiple-comparison-adjusted intervals. Hyperparameters, episode construction,
and the held-out split are frozen before the final sweep. Headline metrics pool
all three rounds. Dose--response points instead use the corresponding $M_r$ or
$B_r$ block and the cumulative artifact snapshot after round $r$.

\section{SafeEvolve}
\label{sec:safeevolve}

\safeevolve is a governance layer around an existing evolution method. It does
not replace the base agent or add a runtime refusal policy. It governs the
persistent skill update at two decision points: writing and reuse.

\subsection{Admission critic and targeted repair}

Before a candidate enters the library, an independent critic audits the full
bundle, including scripts and referenced resources. When a candidate removes a
safety check, assumes authority, broadens a dangerous operation, or embeds an
unsafe tool sequence, the critic produces a targeted repair: restore the check,
add the missing precondition, or narrow the applicability condition. Repaired
skills are re-audited before admission.

\subsection{Verification-preserving evolution}

Skill revisions are constrained to preserve existing verification,
confirmation, authorization, and reversibility requirements unless the update
provides explicit evidence that the requirement is obsolete. This prevents an
efficiency-oriented edit from treating a safeguard as removable overhead.

\subsection{Lineage and provenance}

Each skill stores its source tasks, trajectories, revisions, and exposure
status. Provenance supports attribution and batch rollback when an upstream
experience is later judged unsafe. It also supplies a risk prior to retrieval
and retirement decisions.

\subsection{Safety-aware selection and retirement}

Raw evolution selects skills primarily by utility. \safeevolve augments this
objective with the observed safety impact of reuse. Skills associated with
successful carryover attacks are downweighted, quarantined, repaired, or
retired even when they improve task success. This distinguishes \safeevolve
from capability-only library hygiene, which keeps useful but unsafe skills.

\paragraph{Hermes-native integration.}
Hermes-native evolution enters the same governance interface at its skill-write
approval boundary. In the raw condition, a native candidate follows Hermes's
ordinary approval and storage path. Library-Drift applies capability-oriented
maintenance after writing. Under \safeevolve, the complete native skill bundle
is audited and, when necessary, repaired before approval; accepted skills then
receive the same provenance, safety-aware retrieval, and retirement treatment
as skills produced by agent-agnostic evolution methods. This preserves Hermes's
native authoring loop while changing only the governance of its persistent
updates.

\section{Experimental Setup}
\label{sec:setup}

\subsection{Research questions}

\begin{description}
  \item[RQ1: Safety--utility coupling.] Does skill evolution increase unsafe
  behavior while retaining or improving task utility?
  \item[RQ2: Exposure-to-persistence mechanism.] Does unsafe experience enter
  the skill artifact, get retrieved, and produce post-attack carryover ASR
  while retaining carryover utility?
  \item[RQ3: Scope and attacker budget.] How does misevolution vary across
  methods, agents, task categories and attack strategies, and cumulative doses
  of three, six, and nine malicious tasks?
  \item[RQ4: Mitigation.] Can \safeevolve reduce the three evolved-artifact
  risks, malicious and benign ASR, and carryover ASR while preserving utility?
\end{description}

\section{Discussion}
\label{sec:discussion}

\paragraph{Skills are security-relevant policy.}
The central implication is that a skill library should not be governed as a
passive document store. A stored skill can alter tool choice, ordering,
verification, and authorization across future tasks. Admission, provenance,
rollback, and safety monitoring should therefore be part of the learning
boundary.

\paragraph{Artifact risk and post-attack harm are distinct.}
An evolved artifact can score highly on content unsafety, unsafe
generalization, or stealthiness before its unsafe procedure is retrieved. This
is latent risk, not a safe artifact. Conversely, a runtime policy can block
execution while the persistent library continues to carry the unsafe rule.
Product evaluation should therefore report online utility and ASR, the three
artifact properties, and the two post-attack outcomes separately.

\paragraph{Capability and safety share the same shortcut.}
The difficult cases are not useless malicious skills. They are procedures that
increase success by removing friction that was serving as a safeguard. This is
why disabling evolution loses capability and capability-only library hygiene
does not solve the problem.

\paragraph{Scope and limitations.}
The study focuses on readable skill artifacts produced by CLI and coding agents
and on bounded direct exposure in executable task environments. Results support
claims about evolution-induced incremental risk within this setting. Future
work should test indirect exposure, multimodal skills, opaque learned memories,
longer deployment horizons, interacting libraries, and adaptive attackers.

\section{Conclusion}
\label{sec:conclusion}

Skill self-evolution changes the temporal structure of agent safety. An unsafe
action need not end with the trajectory that produced it: evolution can extract
the successful shortcut, store it as a reusable skill, and apply it later when
the original attack is gone. \gym makes this state transition executable and
replayable; \bench makes it measurable from exposure through artifact formation
and clean-session carryover. \safeevolve shows how to govern the
persistent update while preserving useful learning. Safe self-improvement
requires evaluating not only whether an agent succeeds today, but also what
today's success teaches it to do tomorrow.
